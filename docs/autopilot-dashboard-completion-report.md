## 🎯 架构决策记录

### 1. 为什么使用轮询而非纯 SSE？
**决策**: SSE + 轮询双保险

**理由**:
- SSE 可能因网络问题断开
- 轮询作为备份，确保数据最终一致性
- 不同组件有不同的更新频率需求

### 2. 为什么使用独立页面而非弹窗？
**决策**: 独立路由页面 `/book/:slug/autopilot`

**理由**:
- 监控大盘信息密度高，需要更大空间
- 支持全屏模式，专注监控
- 可以直接分享链接
- 不影响工作台的正常使用

### 3. 为什么使用卡片式而非表格？
**决策**: 卡片式布局（VoiceDrift, Foreshadow, CircuitBreaker）

**理由**:
- 更符合现代 UI 设计趋势
- 响应式布局更容易实现
- 视觉层次更清晰
- 更容易添加交互效果
